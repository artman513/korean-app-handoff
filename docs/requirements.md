# 韓国語学習アプリ「한국어 Study」 機能要件定義書

**バージョン**: 1.3  
**対象**: 開発者向け実装仕様  
**最終更新**: 2026-06-11

---

## 1. 概要

K-POP・韓国アイドルファンをメインターゲットにした韓国語学習アプリ。完全初心者〜中級者が、推し活・コンサート・ファンサイン等の現場で実際に使える韓国語を楽しく継続して習得できることを目的とする。

---

## 2. 対象ユーザー

| セグメント | 説明 |
|---|---|
| メイン | K-POP・韓国アイドル好きの女性（10〜30代）。「推しの言葉を理解したい」「現地で使いたい」が動機 |
| サブ | 韓国語を一から学びたい完全初心者〜中級者（ジャンル問わず） |

## 4. 機能詳細

### F-01 ハングル学習

#### 概要
韓国語の基本文字（ハングル）40音（子音14字＋母音10字＋代表音節）を読み・発音・口の形で習得する。

#### 要件
- 자음（子音）/ 모음（母音）タブで切り替え表示
- 一覧はグリッド表示（4列）。各セルに文字・ローマ字を表示。習得済みセルは視覚的に区別
- セルタップで詳細画面へ遷移
- 詳細画面に表示する情報：文字（大）、ローマ字、口の形のヒント（短文）、例単語、例単語の意味
- 「聞く」ボタンで `SpeechSynthesis`（lang: ko-KR）が例単語を読み上げる
- 読み上げ速度の切り替え：標準（rate: 1.0）/ ゆっくり（rate: 0.7）
- 「習得済みにする」トグルで status を mastered に変更（手動。発音スコア条件は課さない）

#### データ
- ソース：`hangul.json`（静的バンドル）
- 項目：id / char / romaja / type（consonant | vowel | syllable） / mouthHint / exampleWord / exampleMeaning

#### エラー
- `SpeechSynthesis` 非対応 または ko-KR ボイスが取得できない場合：再生ボタンを無効化し「この端末ではお手本再生が使えません」を表示（ボイス一覧は `voiceschanged` イベント後に判定）

---

### F-02 フレーズ音読

#### 概要
推し活・コンサート・ファンサインを中心とした頻出フレーズ50個を、カテゴリ別に一覧表示し、お手本音声を聞いて声に出す練習ができる。

#### 要件
- カテゴリタブ：응원（応援）/ 콘서트 / 굿즈（グッズ）/ 팬싸（ファンサイン） / SNS / 현지（現地） / 인사（あいさつ）
- 各カテゴリにフレーズリストを表示。表示項目：韓国語・日本語訳・ローマ字・難易度バッジ（Lv1〜3）・合格済みマーク
- フレーズタップで詳細画面へ遷移
- 詳細画面：韓国語（大）・日本語訳・ローマ字・例文
- 例文は「例文を見る」タップ時に生成（初回のみGroq、以降キャッシュ）。自動生成はしない（無料枠節約）
- 「お手本を聞く」ボタンで `SpeechSynthesis` が韓国語を読み上げ（標準/ゆっくり切替）
- 「発音してみる」ボタンで F-03（発音チェック）画面へ遷移

#### 例文生成のレスポンス形式（Groqに指示するJSON）
```json
{ "example_ko": "콘서트에서 사랑해요 라고 외쳤어요", "example_ja": "コンサートで愛してると叫びました" }
```
パース失敗時は例文欄を非表示にする（エラー文は出さない）。

#### データ
- ソース：`phrases.json`（静的バンドル、50フレーズ）
- 項目：id / category / ko / ja / romaja / level

### F-03 発音チェック（簡易）

#### 概要
ユーザーの発話を `MediaRecorder` で録音し、Groq Whisper でテキスト化。お手本テキストとの音節一致率でスコアを算出する。不合格時のみGroqが日本語でフィードバックを返す。

#### 対象
- F-02 のフレーズのみ（v1ではハングル単音の発音チェックは行わない。単音は認識精度が安定しないため）

#### 要件
- 対象フレーズを表示（韓国語・日本語訳・ローマ字）
- 「お手本を聞く」→ F-02 と同じ `SpeechSynthesis` で読み上げ（任意）
- 「録音する」ボタンで `MediaRecorder` を起動し音声を録音（audio/webm、Safariは audio/mp4 にフォールバック）
- 録音は最長10秒で自動停止（ボタン再タップでも停止）
- 録音停止後、audio blob を `/api/groq` 経由で Groq Whisper（`whisper-large-v3`、`language: "ko"` を指定）に送信
- Whisperが返したテキストを認識結果として使用
- 認識テキストと正解テキストを音節単位の編集距離で比較しスコア（0〜100）を算出
- スコア表示と判定：
  - 90点以上 → 완벽!（正解演出）
  - 70〜89点 → 좋아요（合格）
  - 69点以下 → 다시!（不合格、フィードバック表示）
- 不合格かつキャッシュ未存在の場合のみ Groq Llama 3.1 8B に「正解文・認識文・ズレた音節を渡し、どの音をどう直すか1〜2文（日本語）」を依頼
- フィードバックは IndexedDB にキャッシュ（key: `feedback:{phraseId}`）
- 合格（70点以上）で SRS 更新（§F-06）。最高スコア・挑戦回数を保存

#### スコア算出
```
score = (1 - levenshtein(正規化済み認識テキスト, 正規化済み正解テキスト) / max(len(認識), len(正解))) × 100
```
正規化：空白・句読点・記号を除去。ズレた音節はUI上でハイライト表示。

#### エラー
- `MediaRecorder` 非対応（極めてまれ）：録音ボタンを無効化しメッセージ表示
- マイク許可が拒否された場合：許可を促すガイドを表示
- Whisperの結果が空・極端に短い（無音や雑音のみ）：スコア計算せず「聞き取れなかったよ、もう一回♡」を表示
- Whisperの結果に韓国語以外（日本語・英語）が大半を占める場合：同上の扱い
- Groq API 失敗・レート制限（429）：「混み合ってるみたい、少し待ってね」を表示しスコア計算を中止

### F-04 パズル学習（骨組み・文型）

#### 概要
動詞カードをドラッグ＆ドロップして文型スロットに当てはめ、正しい活用形の文を完成させる。文型（骨組み）を体感的に習得する。

#### 要件
- 文型カードを表示：文型ラベル（例：〜고 싶어요）・日本語訳
- 出題形式：完成形の選択式。スロットに動詞をドロップすると、その動詞の正しい活用形が候補2〜3個とともに提示され、正しい形を選ぶ（または、動詞ドロップで即完成 → 完成文を表示する簡易モード。実装はまず簡易モードでよい）
- 動詞カードを下部に提示（出題ごとにシャッフル、4枚）：辞書形・日本語訳
- ドラッグ＆ドロップ（dnd-kit使用）。タップでの配置も可能にする（モバイル操作性）
- 正誤判定：配置した動詞の `forms[patternId]`（事前計算テーブル）と完全文字列一致で判定
- 正解時：演出（スタンプ「천재!」「대박!」等）＋完成文を `SpeechSynthesis` で読み上げ
- 不正解時：スロットを揺らすアニメーション＋「다시!」表示（正解は表示しない）
- 連続正解でコンボ演出（3連続〜）：コンボポップアップ＋エフェクト（表示のみ、データ保存しない）
- 1セッション＝文型1つ × 5問。セッション完了で結果サマリ（正答数）を表示し、文型の SRS を更新（§F-06）

#### データ
- `patterns.json`（8文型）：id / label / meaning / template / example / level
- `verbs.json`（15動詞 × 8文型の活用形）：id / dict / meaning / stemType / forms{patternId: 完成文}

#### v1 収録
- 文型8種：〜고 싶어요 / 〜아요어요 / 〜았어요었어요 / 〜아야어야 돼요 / 〜고 있어요 / 〜(으)ㄹ 수 있어요 / 〜(으)ㄹ 거예요 / 안〜아요어요
- 動詞15種：가다 오다 먹다 마시다 보다 사다 하다 자다 읽다 만나다 기다리다 입다 찍다 배우다 살다

### F-06 進捗管理・SRS

#### 概要
学習アイテムごとの進捗を記録し、間隔反復スケジュール（SRS）に基づいて復習タイミングを提示する。

#### SRS更新ルール（機能別）
| アイテム | 更新トリガー | 上げ下げ |
|---|---|---|
| hangul（文字） | 手動「習得済み」トグル | ON: srsLevel=5（mastered固定）/ OFF: 0 |
| phrase（フレーズ） | 発音チェックの結果 | 70点以上: +1 / 69点以下: -1（最低0） |
| pattern（文型） | パズルセッション完了 | 正答率80%以上: +1 / 未満: 変化なし |

- ステータス：new（srsLevel 0）→ learning（1〜4）→ mastered（5）
- SRSレベル（0〜5）で次回復習日を計算

| SRS Lv | 次回復習 |
|---|---|
| 0 | 即時（new） |
| 1 | 1日後 |
| 2 | 3日後 |
| 3 | 7日後 |
| 4 | 14日後 |
| 5 | 30日後（mastered） |

- 進捗画面：習得状況サマリ（ハングル◯/40・フレーズ合格◯/50・文型◯/8）＋復習キュー
- 復習キュー：`nextReviewAt` が現在時刻を過ぎたアイテムを一覧表示、タップで各学習画面へ遷移

#### データ（IndexedDB `progress` ストア）
- 項目：itemId / itemType（hangul | phrase | pattern） / status / srsLevel / nextReviewAt / bestScore / attempts / updatedAt
- bestScore は phrase のみ使用（hangul / pattern は 0 固定で可）

### F-09 ロールプレイAIチャット

#### 概要
K-POPファンの実際のシチュエーションを模したAIとのテキストチャット。返答3択でゲーム感覚で会話練習できる。

#### 要件
- シチュエーション選択画面：영통팬싸（ビデオ通話ファンサイン） / 콘서트 굿즈 구매 / 포카 교환 / 카페에서 주문 / タクシー / 友達と雑談
- 各シチュエーションに難易度（初級/中級）と説明文を表示
- チャット画面：メッセンジャー風UIで会話を進行
  - AIが先に発話（韓国語＋日本語訳）
  - 画面下部に「おすすめ返答3択」を常時提示（韓国語＋日本語訳）。タップで送信
  - 自由入力フォームも併設（韓国語・日本語どちらで入力してもよい。日本語入力時はAIが韓国語で言い直しを提案）
  - タイムスタンプ表示
- 中級モード：AIが「変化球（トラブル・予想外の応答）」を出すオプション
- 영통팬싸モード：タイマー（デフォルト60秒）。残り時間を表示し、0でセッション終了 → ふりかえりへ
- チャット終了後：ふりかえり（ターン数・会話で使った韓国語フレーズの一覧）

#### API設計
- 会話履歴はターンごとに `/api/groq`（Llama 3.3 70B）に全送信
- システムプロンプトで「シチュエーション・AIの役割・難易度・必ずJSONで返す」を指示
- レスポンスJSON形式：
```json
{
  "reply_ko": "어서 오세요! 뭐 드릴까요?",
  "reply_ja": "いらっしゃいませ！何になさいますか？",
  "suggestions": [
    { "ko": "아메리카노 주세요", "ja": "アメリカーノください" },
    { "ko": "추천 메뉴 있어요?", "ja": "おすすめありますか？" },
    { "ko": "잠깐만요", "ja": "ちょっと待ってください" }
  ],
  "end": false
}
```
- `end: true` でAI側から会話終了（店を出る等の自然な区切り）
- パース失敗時：1回だけリトライ。それでも失敗ならそのターンを破棄し「もう一回送ってみて」を表示

#### 回数制限の実装
- 1日のチャットターン数上限：20ターン
- カウントは localStorage `usage.chat = { date: "YYYY-MM-DD", count: n }` で管理。日付が変わったらリセット
- 上限到達時：入力を無効化し「今日の分はおしまい！また明日♡」を表示
- 会話履歴蓄積上限：20ターン（超過時は直近10ターンのみ送信し、それ以前は事前に8Bで要約した1メッセージに置換）

#### エラー
- Groq API 失敗・429：「混み合ってるみたい、少し待ってね」を表示。送信したメッセージは入力欄に復元
- タイムアウト（15秒）：同上

## 6. データ要件

### ローカルストレージ

|---|---|---|
| IndexedDB `progress` | 学習進捗・SRS | F-01〜04, 06 |

- キー形式：`{type}:{itemId}`（例：`feedback:phrase_001`、`example:phrase_001`）
- 同一キーが存在する場合はAPIを呼ばずキャッシュを返す
- Groq API呼び出し前に必ずキャッシュを参照する

### 静的データ（バンドル）
- `hangul.json`（約40件）
- `phrases.json`（50件）
- `patterns.json`（8件）
- `verbs.json`（15件 × 8文型）

## 7. Groq API 利用要件

| 用途 | モデル | 呼び出しタイミング | キャッシュ | 1日の目安 |
|---|---|---|---|---|
| 音声認識（発音チェック） | whisper-large-v3 | 録音完了ごと | なし | 〜発音チェック回数 |
| フレーズの例文生成 | Llama 3.1 8B | 「例文を見る」タップかつキャッシュ未存在 | あり（永続） | フレーズ数上限 |
| 発音フィードバック | Llama 3.1 8B | 不合格かつキャッシュ未存在 | あり（永続） | 〜50回 |
| チャット応答 | Llama 3.3 70B | ターンごと（1日20ターン上限） | なし | 〜20回/ユーザー |
| チャット履歴要約 | Llama 3.1 8B | 履歴が20ターン超過時 | なし | まれ |

- 全Groq呼び出しは Vercel Serverless Function（`/api/groq.ts`）経由（APIキー秘匿）
- `/api/groq` は `task` パラメータ（transcribe | example | feedback | chat | summarize）で処理を分岐
- 無料枠：30 RPM / 6,000 TPM / 1,000 RPD（組織単位）
- OpenAI SDK互換のため `baseURL: "https://api.groq.com/openai/v1"` に差し替えるだけで動作
- レスポンスはJSON形式を指定（`response_format: { type: "json_object" }`）。パース失敗時はフォールバック
- 軽量タスク（例文・フィードバック・要約）は8Bモデルで無料枠を節約。チャットは70Bモデルを使用
- チャットのターン上限を20に設定（6,000 TPMの制約に対応）
- Serverless 側で簡易レートリミット（同一IPから60秒に10回まで）を実装し、APIキーの濫用を防ぐ

## 8. 非機能要件

| 項目 | 要件 |
|---|---|
| 対応ブラウザ | Chrome / Safari / Firefox 最新版（`MediaRecorder` は全主要ブラウザで動作） |
| 対応デバイス | スマートフォン（iOS/Android）＋タブレット＋PC。レスポンシブ対応必須 |
| レイアウト | モバイルファーストで設計。PCはコンテンツを中央寄せ（max-width: 480px）＋左右に余白。Tailwindのブレークポイント（sm/md/lg）を活用 |
| パフォーマンス | 初回ロード3秒以内（Vercel CDN配信前提）、画面遷移1秒以内 |
| アクセシビリティ | 主要ボタン・フォームにaria-label付与。フォントサイズ最小12px |
| セキュリティ | APIキーをフロントに露出しない（Serverless Function経由のみ）。Serverless側にレートリミット |
| データ保持 | 進捗データは端末ローカル（IndexedDB / localStorage）に保存。アカウント機能なし |
| オフライン | 静的コンテンツ（ハングル・フレーズ・パズル）は閲覧/プレイ可。AI機能（発音認識・例文・チャット）はネット必須 |
| コスト上限 | 月$0スタート。Groq 無料枠内で運用。超過時は機能制限（回数上限）で吸収 |

## 9. 技術スタック（前提）

| 領域 | 技術 |
|---|---|
| フロントエンド | Vite + React + TypeScript + Tailwind CSS |
| ルーティング | React Router |
| ホスティング | Vercel（無料枠） |
